# Архитектура мультикуба M03.01 Импорт факт заказов

## Описание архитектуры

### 1. **Источник данных (SQL Database)**
Данные загружаются из MS SQL базы данных из таблицы `CF_Import_By_Orders` с фильтрацией по периодам, заданным в мультикубе `М94.03 Фильтр периодов загрузки Заказов`.

**Загружаемые поля из SQL:**
- `PERIOD` → Период
- `AUFNR` → Заказ
- `KSTAR` → Вид затрат
- `PostDate` → Дата проводки
- `Quantity` → Общее ВведеннКолич
- `UoM` → ВведЕдиницИзмерен
- `MatNum` → Материал
- `MatKText` → Краткий текст материала
- `DocComment` → Название
- `Description` → Описание корреспондирующего счета
- `RepSum` → Знач/ВлтОтч
- `RepCurr` → Валюта отчета
- `LinkDocNum` → № СсылочнДокумента
- `_User` → Имя пользователя
- `DocData` → Дата документа
- `BELNR` → № документа
- `ObjSum` → Сумма/ВлтОб
- `ObjCurr` → Валюта объекта
- `HeadText` → Текст заголовка документа
- `DocDataEntry` → Дата ввода документа
- `DocTime` → Время ввода документа

### 2. **Обработка данных в мультикубе M03.01**

#### **Прямая загрузка (без формул):**
Поля заполняются напрямую из SQL через скрипт импорта.

#### **Расчетные поля (с формулами):**

**A. Маппинг и преобразование данных:**
- **Item.Внутренний заказ** - маппинг из справочника `L01.08 Внутренние заказы` по полю "Заказ"
- **Item.Вид затрат** - маппинг из справочника `L01.07 Виды затрат` по полю "Вид затрат"
- **Item.МВЗ** - получение через маппинг `M97.02 Маппинг Заказ - МВЗ`
- **Item.L04.02 1D Ответственные-Внутренние заказы** - связь ответственных лиц с заказами

**B. Контроль качества данных:**
- **Check Внутренний заказ** - проверка наличия заказа в справочнике (если есть "Заказ", но нет "Item.Внутренний заказ" → ошибка)
- **Check Вид затрат** - проверка наличия вида затрат в справочнике
- **не проставлен ответственный** - проверка наличия ответственного лица
- **Check_Кол-во ошибок** - суммарное количество ошибок
- **Check_Описание ошибки** - текстовое описание найденных ошибок

**C. Фильтры и агрегация:**
- **Есть данные** - проверка наличия данных по периоду
- **NumCount** - подсчет количества записей с данными
- **Есть ошибка** - флаг наличия ошибок в данных

### 3. **Использование данных (Referenced By)**

Данные из M03.01 используются в:

- **M01.04 Ввод данных (по заказам)** - поле 'Ввод без факта' использует данные из M03.01
- **M99.01 MaxNumImport** - поле 'MsxNumOM Заказ' для отслеживания максимального номера импорта
- **М11.02 1D Ответственные-Внутренние заказы** - использует связку ответственных и заказов

### 4. **Вспомогательные компоненты:**

- **М94.03 Фильтр периодов загрузки Заказов** - управляет периодами для загрузки
- **M98.01 Настройки скриптов** - хранит параметры подключения к SQL
- **M99.02 Script logging** - логирование выполнения скрипта
- **M97.02 Маппинг Заказ - МВЗ** - справочник соответствия заказов и МВЗ
